### AI Adoption Is Not Optional

Donovan framed AI-assisted development as a company-wide imperative, not an opt-in experiment. The models are now capable enough that poor results are more likely a skill issue than a tool limitation. He compared the discomfort to learning a new programming language -- pushing through the initial friction is how you unlock fluency.

### Plan-Execute-Verify Framework

The core development workflow Donovan wants everyone to internalize:

- **Plan** -- Spend more time than feels natural. Better plans directly produce better AI output. Reduces variability.
- **Execute** -- Trust the tools more. Sit in the discomfort longer before giving up. This is where most people struggle.
- **Verify** -- Automate verification (tests, linting, builds). AI can self-check if given the right infrastructure. Human review remains critical -- everyone becomes QA.

### Identity Shift

- Developers may experience an "identity crisis" as code writing shifts to AI
- Reframe identity: problem solver, planner, communicator -- not code typist
- Core engineering principles haven't changed; the medium has
- "My favorite engineers are the ones that just like to solve problems and it doesn't really matter how" -- Donovan

### CLAUDE.md and Continuous AI Learning

- Set up a CLAUDE.md file in every project as the lowest barrier entry point
- When Claude repeats a mistake, teach it and add a rule so it doesn't happen again
- If it breaks a rule, ask it _why_ -- find conflicting or unclear rules
- Let Claude write the rules for you as part of leaning into AI workflows

### AI Platform Core Team

- Will grow over time

### Pedro's AI Journey

- Started October 2024 with ChatGPT (a chatbot-only camera app) -- "it was terrible"
- Early 2025: still skeptical, didn't even use Copilot autocomplete
- Today: writes nearly all code through Claude, stays in the loop ("vibing engineering, not vibe coding")
- Productivity gain is mostly in _quality_, not raw output speed
- AI helps navigate unfamiliar codebases and frameworks by removing the friction of recursive learning
- Good engineering practices create a virtuous cycle: good standards help AI, AI helps enforce higher standards
- Reduced procrastination -- getting unstuck is nearly frictionless now
